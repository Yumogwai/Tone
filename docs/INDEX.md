# Tone — Documentation Hub

Start here. All project docs live in `docs/` as plain Markdown and link from this index.
Use standard Markdown links (`[label](file.md)`), not wikilinks.

## Core docs

- [Status](STATUS.md) — current phase, what's done, what's next (the living state file)
- [Product Overview](PRODUCT.md) — what Tone is, who it's for, the design language
- [Plans](PLANS.md) — future/aspirational work (kept separate from shipped reality)

## Conventions

- Docs describe **what is shipped now**, not plans. Plans go in `PLANS.md`.
- When a code change makes a doc inaccurate, update the doc in the same commit.
